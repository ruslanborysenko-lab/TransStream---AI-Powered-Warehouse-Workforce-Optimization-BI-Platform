3. MATHEMATICAL WORKFORCE FORMULAS
   - **Loader Formula**: 
     Total_Hours = (Direct_20 + Direct_40) √ó 3h + (Cross_20 + Cross_40) √ó 5h
     Loader_Count = MAX(2, (Total_Hours √∑ 160) √ó 4)
     Example: May had 148h ‚Üí 4 loaders (was 10, optimized 60% reduction!)
   
   - **Forklift_Operator Formula**:
     Pallet_Hours = Pallet_Direct √ó 1h + Pallet_Cross √ó 2h
     Operator_Count = MAX(1, Pallet_Hours √∑ 160)
     Example: May had 259h ‚Üí 2 operators (was 0, added role!)
   
   - **Operation_manager Formula**:
     Office_Ops = Other_revenue + Reloading + Storage + Additional
     Manager_Count = MAX(2, MIN(5, Office_Ops √∑ 150))
     Example: May had 510 ops ‚Üí 3 managers (was 2, +50% increase!)

3. MATHEMATICAL WORKFORCE FORMULAS
   "I developed 3 mathematical formulas for workforce optimization based on operation types 
   and time requirements. For Loaders: Manual_Ops_Hours = (Direct_20 + Direct_40) √ó 3h + 
   (Cross_20 + Cross_40) √ó 5h, then Loader_Count = MAX(2, Hours √∑ 160 √ó 4). The '√ó4' is 
   because manual operations need 4-person teams. The MAX(2, ...) ensures safety minimums. 
   For Forklift_Operators: Pallet_Hours = Pallet_Direct √ó 1h + Pallet_Cross √ó 2h, then 
   Operator_Count = MAX(1, Hours √∑ 160). For Operation_managers: Office_Ops = sum of 4 
   service types, then Manager_Count = MAX(2, MIN(5, Ops √∑ 150)). The MIN(5, ...) caps 
   managers since office work has diminishing returns. These formulas are business-rule 
   driven, not arbitrary."

4. CASCADE FORECASTING STRATEGY
   "The forecasting is more sophisticated than standard linear regression. I use cascade 
   forecasting where each month depends on the previous: October uses May-Sept data, 
   November uses May-Oct (including October's prediction), December uses May-Nov. This 
   captures momentum - if operations are growing in October, November prediction reflects 
   that trend. I train 10 separate LinearRegression models (one per operation type) using 
   month numbers as features (May=5, June=6, etc.). For employee forecasting, I don't use 
   regression directly - instead, I calculate the operations change ratio from September 
   baseline and scale employees with role-specific sensitivities. Loaders have 60% growth 
   sensitivity (manual work scales fast), while managers have 40% (office work has economies 
   of scale)."

5. SESSION STATE MANAGEMENT COMPLEXITY
   "Managing state in Streamlit is tricky with 1,369 lines of code and multiple workflows. 
   I use 5 state variables: show_optimization (Boolean), optimization_data (cached AI 
   response), show_forecast (Boolean), forecast_data (tuple of DataFrames), and user 
   selections (operation, month). The flow is: user clicks 'Optimize' ‚Üí AI processes ‚Üí 
   optimization_data stored ‚Üí show_optimization=True ‚Üí optimization section renders. Then 
   'Forecast' button ‚Üí forecast_data computed ‚Üí show_forecast=True ‚Üí forecast section 
   renders. User selections (operation, month) trigger executive dashboard. The challenge 
   is cache invalidation - when user re-optimizes, I clear forecast_data and 
   show_forecast=False to force recalculation since forecasts depend on optimization. 
   Without careful state management, users would see stale data or conflicting results."

6. AI RESPONSE PARSING ROBUSTNESS
   "Parsing AI output is non-trivial because GPT-4o's format can vary. I implemented a 
   multi-strategy parser: First, split by newlines and whitespace. Second, validate each 
   row - check if first element is a valid month name (May/June/...), ensure ‚â•15 columns, 
   filter out header rows. Third, handle two formats - some responses have 16 columns 
   (with month), others have 15 (without month, need to infer). Fourth, fallback to original 
   data if parsing completely fails. I also validate data types - convert all numbers with 
   pd.to_numeric(errors='coerce') and fillna(0) for safety. This robustness is critical 
   for production - the system doesn't crash, it gracefully degrades."

7. EXECUTIVE DASHBOARD DESIGN
   "The executive dashboard is designed for non-technical stakeholders. I chose 4 KPIs that 
   executives care about: Selected Operation (drill-down metric), Total Operations (overall 
   activity), Total Staff (cost metric), and Productivity (efficiency ratio). Each shows 
   month-over-month delta as percentage - green for growth, red for decline. The dashboard 
   has 4 charts: All Operations pie (big picture), Operation Share donut (context), Staff 
   Distribution bar (team composition), Month Comparison bar (trend). I use color psychology - 
   gray for previous month (neutral), green/red for current month (signal). The productivity 
   metric (ops/employee) is a game-changer - it shows efficiency, not just headcount. If 
   operations grow 20% but staff grows 30%, productivity declined - this KPI catches that."
